# Plan: Complete dashboard server-function → REST API migration

## Context
The project is migrating dashboard pages away from TanStack `createServerFn` / `useServerFn` toward plain REST API endpoints under `/api/dashboard/*`. The previous session created the dashboard auth helper, the dashboard router, wired it in `server.ts`, and created a client helper (`api-client.ts`). It also migrated `overview.tsx` and `venom-models.tsx`. The session was interrupted before finishing the remaining pages that still call `venom.functions.ts`.

## Current state
- `src/lib/dashboard-auth.server.ts` — exists, validates `Authorization: Bearer <token>`.
- `src/lib/api/dashboard-router.server.ts` — exists, exposes 15 endpoints (metrics, providers, venom-models, api-keys, routing-rules, account-models, quotas).
- `src/server.ts` — wires `/api/dashboard/*` before TanStack SSR.
- `src/lib/api-client.ts` — `api.get/post/patch/delete` helper that injects the Supabase Bearer token.
- Migrated pages: `overview.tsx`, `venom-models.tsx`, `quota.tsx` (verified, no imports from `venom.functions.ts`).
- Pages still importing from `venom.functions.ts`:
  - `src/routes/_authenticated/routing.tsx`
  - `src/routes/_authenticated/api-keys.tsx`
- Other `useServerFn` usages (providers, models, etc.) are from `integrations.functions.ts` and are **out of scope** for this task.

## Goal
Finish the migration by converting `routing.tsx` and `api-keys.tsx` to use `api-client.ts` instead of `useServerFn` and `venom.functions.ts`. Keep the DB layer untouched and preserve all existing UI behavior.

## Implementation steps

### Step 1 — Update `src/routes/_authenticated/routing.tsx`
- Remove `import { useServerFn } from "@tanstack/react-start"`.
- Remove the `venom.functions.ts` import block:
  - `listRules`
  - `listApprovedAccountModels`
  - `createRoutingRule`
  - `deleteRoutingRule`
  - `toggleRoutingRule`
- Add `import { api } from "@/lib/api-client"`.
- Replace query/mutation functions:
  - `listRules()` → `api.get<RoutingRule[]>("/api/dashboard/routing-rules")`
  - `listApprovedAccountModels()` → `api.get<ApprovedModel[]>("/api/dashboard/account-models")` (define `ApprovedModel` type from existing usage or infer from endpoint).
  - `createRoutingRule({ data: body })` → `api.post<RoutingRule>("/api/dashboard/routing-rules", body)`
  - `toggleRoutingRule({ data: { id, active } })` → `api.patch<RoutingRule>(`/api/dashboard/routing-rules/${id}`, { active })`
  - `deleteRoutingRule({ data: { id } })` → `api.delete<{ success: true }>(`/api/dashboard/routing-rules/${id}`)`
- Keep query keys, invalidation, optimistic updates, loading/error UI exactly as-is.
- Preserve types (e.g. `RoutingRule`) already imported from `@/lib/db/venom.server`.

### Step 2 — Update `src/routes/_authenticated/api-keys.tsx`
- Remove `import { useServerFn } from "@tanstack/react-start"`.
- Remove the `venom.functions.ts` import block:
  - `listApiKeys`
  - `createApiKey`
  - `revokeApiKey`
  - `deleteApiKey`
- Add `import { api } from "@/lib/api-client"`.
- Replace query/mutation functions:
  - `listApiKeys()` → `api.get<KeyRowData[]>("/api/dashboard/api-keys")`
  - `createApiKey({ data: body })` → `api.post<{ raw_key: string; key: KeyRowData }>("/api/dashboard/api-keys", body)`
  - `revokeApiKey({ data: { id } })` → `api.patch<KeyRowData>(`/api/dashboard/api-keys/${id}`)`
  - `deleteApiKey({ data: { id } })` → `api.delete<{ success: true }>(`/api/dashboard/api-keys/${id}`)`
- Keep the existing UI, toast messages, copy-to-clipboard flow, and query invalidation.
- Preserve the `KeyRowData` type by aliasing the API response type or keeping a local type alias.

### Step 3 — Verify and clean up
- Run `grep -r "from \"@/lib/venom.functions\"" src/` to confirm no page/component still imports from `venom.functions.ts`.
- If `venom.functions.ts` is no longer imported anywhere in `src/`, consider deleting it or leaving it untouched per the user's prior direction to avoid unrelated changes. **Recommendation:** leave it in place unless explicitly asked to remove it, to keep the diff minimal.
- Run TypeScript check: `npx tsc --noEmit`.
- Run lint: `npm run lint` (or `npx eslint src/routes/_authenticated/routing.tsx src/routes/_authenticated/api-keys.tsx`).
- Run dev server and manually verify:

  - Creating, toggling, and deleting a rule works with plain JSON in the Network tab.
  - `/api-keys` loads keys.
  - Creating, revoking, and deleting a key works with plain JSON in the Network tab.
